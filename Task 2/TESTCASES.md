## 1. QA-intership/api/1/item/{id}
## 1.1 Post https://qa-internship.avito.com/api/1/item - Сохранение объявления
**Headers:**
| **Content-Type** | **Accept** |
|--------------|--------|
| application/json | application/json|


Body:  
Method: row 
```
{  
  "sellerID": "<integer>",  
  "name": "<string>",  
  "price": "<integer>",  
  "statistics": {  
    "likes": "<integer>",  
    "viewCount": "<integer>",  
    "contacts": "<integer>"  
  }  
}
```
- sellerID - integer (диапозон 111111 - 999999)
- name - string (более 2990 символов)
- price - integer (макс. 19 символов)
- likes - integer (макс. 19 символов)
- viewCount - integer (макс. 19 символов)
- contacts - integer (макс. 19 символов)

**Ожидаемый результат:** 200 OK  
```
{
  "id": "<string>",
  "sellerId": "<integer>",
  "name": "<string>",
  "price": "<integer>",
  "statistics": {
    "likes": "<integer>",
    "viewCount": "<integer>",
    "contacts": "<integer>"
  },
  "createdAt": "<string>"
}
```
**Примечание:** При отправке запроса поле id принимает знасение integer, в документации при выводе указано, что поле id принимает значение string.

**Ожидаемый результат:** 400 Bad request  
```
{
  "result": {
    "messages": {
      "culpa_b92": "<string>",
      "enim_24f": "<string>",
      "mollit_aa": "<string>"
    },
    "message": "<string>"
  },
  "status": "<string>"
}
```

## 1.2.4 Получить объявление по идентификатору - ошибка валидации и экранирования
| Param | Value_1 | Value_2 |
|--------|--------|--------|
| id | fb32а45f-b394-402а-ас54-е1е62bb99871 | fb32а45f-b394-402а-ас54-е1е62bb9987! |

## Шаги воспроизведения:
- Ввести в параметрах поля id созданный через метод Post идентификатор объявления
- Value_1: Заменить омоглифы (буквы, схожие по написанию) на кириллицу
- Value_2: Заменить омоглифы (буквы, схожие по написанию) на кириллицу и заменить последнюю единицу на специальный символ из вариантов: "!", "*" "(", ")".
---
## Фактический результат:
Response_1:  
400 Bad Request  
Body_1:  
```
{
    "result": {
        "message": "ID айтема не UUID: fb32а45f-b394-402а-ас54-е1е62bb99871",
        "messages": {}
    },
    "status": "400"
}
```
Response_2:  
400 Bad Request  
Body_2:  
```
    "result": {
        "message": "ID айтема не UUID: fb32%D0%B045f-b394-402%D0%B0-%D0%B0%D1%8154-%D0%B51%D0%B562bb9987!",
        "messages": {}
    },
    "status": "400"
}
```
---
## Комментарий
**Response_1:** Отработка ошибки корректная. Кириллица в ID просто кодируется в UTF-8, что является нормальным поведением и не влияет на результат.  
**Response_2:** Сервис некорректно обрабатывает спецсимвол ! в ID — в сообщении об ошибке строка искажается и отображается в виде закодированных символов. Вероятно, проблема в механизме экранирования. Баг заведен в файле [BUGS.md](/Task%202/BUGS.md).
